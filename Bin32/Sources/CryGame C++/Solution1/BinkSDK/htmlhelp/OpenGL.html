OpenGL

Discussion

OpenGL is an industry standard 3D graphics API that is used drive your 3D graphics card.

On Windows, you can check out the Exam3D.c file and the GL support file ("GLRAD3D.C") for an example of 3D video textuer playback. You can also use OpenGL to do fast alpha-plane based translucent video playback (see "ExamAlph.C" and "GLRAD3D.C" for an example). Our Windows OpenGL sample code only uses the fixed function pipeline. To use GL pixel shaders (which are the best way to display 3D video), you need to write your own GLSL code, or use Direct3D with our example code ("BinkTextures.cpp").

On Sony Playstation 3, you use OpenGL with cG pix shaders to display your video frames - this is fastest and best quality option.

You can read more about OpenGL at the GLSetup website.

Note that we provide a cross platform API to do pixel shader blitting on Windows, Sony PS3, Wii, GameCube, Xbox and Xbox 360. See the BinkTextures.h file for details.


IMPORTANT NOTE! Remember that when playing non-alpha plane videos through the 3D system, you must turn off the alpha blending renderstate! Bink doesn't set the alpha value on non-alpha pixel formats (like BINKSURFACE32) because it would slow blitting down! If you try to play non-alpha video and the screen is black, then you probably have the alpha render state accidentally turned on.
