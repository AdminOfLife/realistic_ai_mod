Bink Examples for Win64

Discussion

The Bink SDK supplies several example programs for Win64. To create these examples in an IDE-based environment, just create a new project and include the appropriate C source file (ExamWin.C, ExamMSS.C, etc), and then add the Bink for Windows import library to the project (BinkW64.LIB).

The modern examples of Bink playback are:

For a simple example using 2D blitting and background decompression, see the "ExamWinThr.C" file.

For a simple example using DX9 pixel shaders and background decompression, see the "Exam3DPSThr.CPP" file.

This example uses a high-level wrapper to D3D that is compatible with the original Xbox, the Sony PS3, and the Xbox 360 - see the "BinkTextures.h" and the "BinkTextures.cpp" files for details.

For an example of using the Miles Sound System, see the "ExamMSS.C" file.

The following is a list of other examples:

For a simple example of using 2D blitting and foreground decompression, see the "ExamWin.C" file.

For an example of using using DirectDraw and foreground decompression, see the "ExamDDP.C" file.

For an example of using Bink with pixel shaders to do the YUV color conversion, see the "Exam3Dps.CPP" file.

For an example of using Bink to play an video with an alpha channel using pixel shaders, see the "Exam3Dpsa.CPP" file. This example uses the same support files as Exam3Dps.CPP.

For an example of using Bink with a fixed function 3D pipeline, see the "Exam3D.C" file. This example uses a high-level wrapper to the fixed function pipeline - it can be compiled to use DirectX 9, DirectX 8, DirectX 7, or OpenGL. See the "RAD3D.h" "DX9RAD3D.CPP", "DX8RAD3D.CPP", "DX7RAD3D.CPP", or "GLRAD3D.C" files for more details.

For an example of using Bink to play an video with an alpha channel using a fixed function 3D pipeline, see the "Exam3Da.CPP" file. This example uses the same support files as Exam3D.C.
