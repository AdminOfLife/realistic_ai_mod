Drawing the Bink Video Frames on the Xbox 360

Discussion

Video playback on the Xbox 360 is pretty easy compared to a PC - the hardware is damn fast and, most importantly, it is constant. If it plays back perfectly on your dev kit, then it will on everyone else's too!

For playback on Xbox 360, you use pixel shaders to do the YUV to RGB colorspace conversion. You do this by having Bink decompress directly into Direct3D 8-bit luminance textures (two sets of 3 of them - one for Y, cR, cB). Then, you simply render after calling BinkDoFrame. This requires the use of the BinkGetFrameBuffersInfo and BinkRegisterFrameBuffers functions. We've supplied the shaders and all the code to render them in the BinkTextures.CPP file, and the ExamXenon.CPP file shows how to draw them.

We provide a cross platform API to do pixel shader blitting on Windows, Sony PS3, Wii, GameCube, Xbox and Xbox 360. See the BinkTextures.h file for details.

If you need to use an alpha plane, then Bink still has you covered. We've supplied a pixel shader to read and blend the alpha planes too. See the ExamXenA.CPP to see how to display an alpha video.

The Xbox 360's 3D hardware is terrific - you can display HDTV video with no CPU hit for color conversion at all.
