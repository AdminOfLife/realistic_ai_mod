BinkSoundUseNDSSound

The function BinkSoundUseNDSSound tells Bink to use the Nintendo DS sound hardware as its sound output system.

S32 BinkSoundUseNDSSound(
  U32 channels_and_alarms
);

In Parameters

channels_and_alarms
a bitfield value that tells Bink which sound channels and sound alarms it can use.

Returns

Returns whether Bink successfully switched to DS sound as its output layer (non-zero for success, zero for failure).

Discussion

This function (macro actually) tells Bink to use the Nintendo DS sound hardware as its audio output.

This function is global - you only need to call it once. You must, however, call it before you call BinkOpen for the first time, or the function will fail.

The channels_and_alarms parameter tells Bink what sound channels and alarms it may use. Bink will only use these resources while a movie is playing. For every Bink movie that is playing simultaneously, you need at least one sound channel (two for stereo), and one sound alarm. Just choose ones that your game won't use while a movie is playing. To specify the bit field values, you use two simple helper macros. For example, if you wanted Bink to use channel 4 and alarm 1, you use: 


BinkSoundUseNDSSound( BINK_PACK_CHANNEL( 4 ) | BINK_PACK_ALARM( 1 ) );

If you wanted Bink to use channels 4 and 5 for stereo output and alarm 2, you'd use: 


BinkSoundUseNDSSound( BINK_PACK_CHANNEL( 4 ) | BINK_PACK_CHANNEL( 5 ) | BINK_PACK_ALARM( 2 ) );
