Drawing the Bink Video Frames on the Nintendo DS

Discussion

Video playback on the Nintendo DS is pretty easy compared to a PC - the hardware is constant and simple. If it plays back perfectly on your dev kit, then it will on everyone else's too!

Since you are fairly close to the DS metal, you have a bunch options for blitting:

1) You can have Bink directly convert from YUV right into a single RGB video buffer. This option is very fast and uses the least memory, but the video can tear, since you are blitting directly into the memory that is being used to draw the screen. On many videos, this is not noticable, so it's an option you can definitely try.

2) You can have Bink directly convert from YUV right into multiple RGB video buffers and then use page flipping. This uses little memory (just one more buffer than option one), but it does mean that Bink has to blit the entire video frame for each frame (because there are multiple buffers, we can't simply use deltas from the previous frame).

3) You can have Bink blit into an off-screen buffer, and then use the DMA system to send it up to the video display. This is as fast as option 1 only it won't have video tearing. It does use one extra buffer for blitting though. This is the option that the Bink example (BinkNDS.c) uses.

4) You can also use 16-bit pixels or 8-bit palettized pixels. 16-bit looks a lot better, and the hardware is actually natively 16-bit, so we recommend just stick with BINKSURFACE555 or BINKSURFACE5551. 8-bit is really only useful when off-screen buffering and you are tight on memory (because the buffer would be half size).
