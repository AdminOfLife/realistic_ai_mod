Alpha Planes

Discussion

Alpha planes are bitplanes that describe what is and what isn't transparent or translucent on a video frame. They are much more useful than simple color keying because you control the transparency right to the pixel - no more weird problems when the color key blends into the edges of the non-transparent pixels!

You create alpha planes with a video editing package like Adobe Premiere, or by telling your 3D package to generate alpha planes as it renders the video frames.

There are two ways to store alpha data with the pixels - premultiplied alpha and normal alpha. You should always use premultiplied alpha - it's faster, it compresses better, and it will be filtered correctly on 3D hardware.

At compression time, you can tell Bink to include the alpha plane with no change, convert it to premultiplied alpha, or just convert to premultiplied alpha during filtering (which is good if you are resizing the video, but don't want to use premultiplied). Make sure you don't select premultiplied alpha if the incoming alpha plane is already premultiplied - otherwise the video will be much too dark.

Playing back alpha planes is a little tricky because applying alpha transparency is fairly expensive CPU-wise. To keep performance up, we recommend using your 3D video card to merge bitmap using alpha planes.

Harnessing your 3D video card means that you must use OpenGL, Direct3D, or your console's 3D library to talk to your video card.

On Win32 and Win64, Bink includes an example that can play alpha translucent video for Direct3D using pixel shaders ("Exam3DPSa.CPP"). This example uses an example source code that you should be able to just drop into your application ("BinkTextures.h" and "BinkTextures.CPP").

Also on Win32 and Win64, Bink includes an example that can play alpha translucent video using the 3D fixed function pipeline for both OpenGL and Direct3D ("Exam3Da.C"). To use OpenGL, or Direct3D, you just link to the appropriate 3D interface file ("GLRAD3D.C", "DX9RAD3D.CPP', "DX8RAD3D.CPP', or "DX7RAD3D.CPP').

On Xbox, Bink include an example that uses Direct3D to play alpha translucent video (ExamXAlf.CPP). You'll also need the Xbox 3D interface file, "BinkTextures.CPP".

On Xbox 360, Bink include an example that uses Direct3D to play alpha translucent video (ExamXenA.CPP). You'll also need the Xbox 360 3D interface file, "BinkTextures.CPP".

On Sony PS3, Bink include an example that uses PSGL to play alpha translucent video (ExamPS3A.CPP). You'll also need the GL 3D interface file, "PSGLTextures.CPP".

On Sony PSP, Bink include an example that uses libGu to play alpha translucent video (ExamPSP.CPP). You'll also need the Gu 3D interface file, "BinkGuTextures.CPP".

On Nintendo Wii, Bink include an example that uses the GPU to play alpha translucent video (ExamWiiTex.C). You'll also need the Wii 3D interface file, "WiiTextures.C".

On Nintendo GameCube, Bink include an example that uses the GPU to play alpha translucent video (ExamNGCtex.C). You'll also need the GameCube 3D interface file, "NGCTextures.C".

Note that we provide a cross platform API to do pixel shader/register combiner alpha blitting on Windows, Sony PS3, Wii, GameCube, Xbox and Xbox 360. See the BinkTextures.h file for details.


IMPORTANT NOTE! Remember that when playing non-alpha plane videos through the 3D system, you must turn off the alpha blending renderstate! Bink doesn't set the alpha value on non-alpha pixel formats (like BINKSURFACE32) because it would slow blitting down! If you try to play non-alpha video and the screen is black, then you probably have the alpha render state accidentally turned on.
