Bink SDK 1.9c

Direct3D

Discussion

Direct3D is a Microsoft supplied 3D graphics API that is used to drive your 3D graphics chip. There have been lots of versions of D3D (which is both a good and a bad thing), so we have several support examples.

On Win32 and Win64, you can display video using Direct3D using the fixed function pipeline or pixel shaders. Pixel shaders are the fastest option, but aren't supported on all 3D hardware. The fixed function pipeline support works on all 3D hardware but is slower and doesn't look quite as nice.

You can see a pixel shader example in the "Exam3Dps.CPP" file (which also uses the "BinkTextures.h" and "BinkTextures.CPP" interface files).

For fixed function support, you can check out the Exam3D.c file and one of the fixed function support files ("DX9RAD3D.CPP", "DX8RAD3D.CPP" or "DX7RAD3D.CPP").

You can also use the Direct3D support (both pixel shaders and fixed function) to do fast alpha-plane based translucent video playback (see "Exam3DPSa.CPP" or "Exam3Da.C" for example source code).

You will usually use Direct3D to display your video frames on the Xbox, since it can render right out of the YUV textures. The Xbox supports linear, non-power-of-2 textures, and pixel shaders, so this is fast and super-high quality.

You will always use pixel shaders to display your video frames on the Xbox 360 (and Sony Playstation 3, actually) - this is fastest and best quality option.

You can read more about Direct3D at the Microsoft DirectX website.

Note that we provide a cross platform API to do pixel shader blitting on Windows, Sony PS3, Wii, GameCube, Xbox and Xbox 360. See the BinkTextures.h file for details.


IMPORTANT NOTE! Remember that when playing non-alpha plane videos through the 3D system, you must turn off the alpha blending renderstate! Bink doesn't set the alpha value on non-alpha pixel formats (like BINKSURFACE32) because it would slow blitting down! If you try to play non-alpha video and the screen is black, then you probably have the alpha render state accidentally turned on.
