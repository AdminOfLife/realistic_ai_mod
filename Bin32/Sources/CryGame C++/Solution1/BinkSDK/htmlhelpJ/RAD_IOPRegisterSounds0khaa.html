RAD_IOPRegisterSoundCallback

関数 RAD_IOPRegisterSoundCallback は、サウンド データ構造体を登録して、ユーザーのサウンド インタラプト コールバックを呼び出します。

S32 RAD_IOPRegisterSoundCallback(
  RAD_IOP_SOUND_DATA *sound_data
);

出力パラメータ

sound_data
サウンド コールバック情報を指定します。

戻り値

サウンド コールバックが登録された場合は 0 以外の値、失敗した場合は 0 が返されます。

説明

この関数は、サウンド コールバック構造体を登録して、サウンド インタラプト コールバックを呼び出します。サウンド コールバックは、実際には RAD_IOPStartSound の呼び出しまで開始しません。

この構造体は、サウンド コールバックが RAD_IOPUnregisterSoundCallback 関数で登録解除されるまで、有効にしておく必要があります。

サウンド コールバック関数は、インタラプトで呼び出されるため、コードは必ずインタラプト セーフにする必要があります。

サウンド コールバックが実行されるたびに、sound_data 構造体の address および length フィールドで指定されたバッファにサウンド データを上書きするか、ミックスする必要があります。sound_data 構造体の overwrite フラグは、データを直接バッファにコピーする必要があるか (overwrite が TRUE)、またはデータを直接バッファにミックスする必要があるか (overwrite が FALSE) を示します。ミックスするには、S16 値をすでにバッファにある S16 値に加算します (-32768 〜 32767 に制限されます)。

通常、バッファは 48,000 Hz、16 ビットのステレオ PCM データ ブロックで、256 の左側サンプルの後ろに 256 の右側サンプルが続くソニー形式です。長さは常に 512 サンプルの倍数です (現在のバージョンの RAD_IOP では、2,048 バイト)。RAD_IOP_SPDIF + RAD_IOP_NEED_CORE0 フラグが使用されている場合、EE から返されるデータ ブロックは、RAW ビットストリーム データとして扱われ、DTS レシーバに送られます (RAD_IOP_SPDIF モードではコールバックの overwrite フラグに意味はありません)。
