Bink SDK 1.9c

Direct3D

説明

Direct3D は、3D グラフィック チップを制御するために使用される 3D グラフィック API として、Microsoft から提供されています。D3D には多くのバージョンがあるので (それにはよい面と悪い面がありますが)、サポート サンプルも複数用意されています。

Win32 と Win64 では、固定機能パイプラインまたはピクセル シェーダを使用して Direct3D でビデオを表示できます。ピクセル シェーダのほうが高速ですが、サポートされていない 3D ハードウェアもあります。固定機能パイプラインはすべての 3D ハードウェアでサポートされますが、低速で、品質もピクセル シェーダほどよくありません。

ピクセル シェーダのサンプルは、Exam3Dps.CPP ファイルにあります (BinkTextures.h および BinkTextures.CPP インターフェイス ファイルも使用されます)。

固定機能サポートは、Exam3D.c ファイルと固定機能サポート ファイル (DX9RAD3D.CPP、DX8RAD3D.CPP、DX7RAD3D.CPP) の 1 つで確認できます。

また、Direct3D サポート (ピクセル シェーダと固定機能の両方) を使用して、高速アルファ平面に基づく透過ビデオ再生を行うことができます (ソース コードのサンプルについては、Exam3DPSa.CPP または Exam3Da.C を参照)。

Xbox では、YUV テクスチャから直接レンダリングできるため、通常、Direct3D を使用してビデオ フレームを表示します。Xbox は、リニアな非 2 乗テクスチャとピクセル シェーダをサポートするため、高速で超高品質です。

Xbox 360 では (実際はソニー プレイステーション 3 でも)、常にピクセル シェーダを使用してビデオ フレームを表示します。これが最も高速で高品質なオプションです。

Direct3D の詳細については、Web サイト Microsoft DirectX を参照してください。

Windows、ソニー PS3、Wii、ゲームキューブ、Xbox、および Xbox 360 でピクセル シェーダ ブリッティングを実行するためのクロス プラットフォーム API が用意されています。詳細については、BinkTextures.h ファイルを参照してください。


重要事項 3D システムで非アルファ平面ビデオを再生する場合は、アルファ ブレンディングのレンダリング状態をオフにする必要があります。Bink は、非アルファ ピクセル フォーマット (BINKSURFACE32 など) でアルファ値を設定しません。設定すると、ブリッティング速度が遅くなるためです。非アルファ ビデオを再生しようとして、画面が黒くなる場合は、アルファのレンダリング状態がオンになっている可能性があります。
