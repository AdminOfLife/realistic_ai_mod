Win32 での 2D ハードウェア ブリッティングの利点と欠点

説明

2D ハードウェア YUV オーバーレイの利点

1) 超高速です。画面にブリットしなくても、サーフェスに描くだけでピクセルが表示されます。

2) クリッピングが組み込まれています。

3) ウィンドウ上にフロートするので、アプリケーションの邪魔になりません。

4) CPU に負荷をかけずに縮小および拡大できます。
2D ハードウェア YUV オーバーレイの欠点

1) ハードウェアであることです (ビデオ カードとその DirectDraw ドライバに左右されます)。

2) ビデオ カードによって表示が異なることがあります (特に拡大したとき)。

3) ウィンドウ上にフロートするので、アプリケーションに完全に統合したように表示することは困難です。

4) Direct3D を実行している間は、原則として使用できません。


2D ハードウェア YUV オフスクリーン バッファの利点

1) 高速です (ハードウェアで YUV 変換を処理する場合。一部の DirectDraw ドライバはホストの CPU で変換を処理します)。

2) 通常、自由にクリッピングを実行できます。

3) 基本的には標準のブリッタ (標準的な古いオフスクリーン ブリッタとまった同じで、多くの人が使用法を知っている) と同じように動作します。

4) CPU に負荷をかけずに縮小および拡大できます。
2D ハードウェア YUV 変換ブリッタの欠点

1) これもハードウェアに組み込まれているので、ビデオ カードとドライバが正常に動作しているかどうかに影響されます。3D では、これはまれです。

2) ソフトウェアで変換されることがありますが、通知されません。

3) ビデオ カードによって表示が異なることがあります (この場合も、特に拡大したとき)。
