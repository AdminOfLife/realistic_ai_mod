RAD_IOP_ASYNC_OPEN_DATA

構造体 RAD_IOP_ASYNC_OPEN_DATA は、RAD_IOPQueueAsyncOpenFile 関数で使用される構造体です。

struct RAD_IOP_ASYNC_OPEN_DATA
{
  CHAR_PTR name;
  RAD_IOP_OPENCALLBACK callback;
  U32 group;
  RAD_IOP_STATUS_VALUES status;
  S32 file_handle;
  U32 size;
};

入力メンバ

name
開くファイルの名前を指定します。名前の前に「cdrom0:」、「host0:」、または「hdd0:」が必要です。CD または DVD ファイルの場合は、ファイル名の末尾が「;1」である必要があります。
callback
終了時に実行するコールバックを指定します。コールバックが不要な場合は、このフィールドに 0 を設定します。
group
オプションです。このオープンのグループ ID を指定します。グループ ID を使用すると、グループに含まれるすべての要求の I/O を必要に応じて取り消すことができます。

出力メンバ

status
非同期オープンの現在のステータスを示します。
file_handle
status として RAD_IOP_STATUS_DONE が返された場合、開いているファイルのハンドルを返します。
size
開いているファイルのサイズを返します。

説明

この構造体は、RAD_IOPQueueAsyncOpenFile 関数の入力と出力の両方で使用されます。入力側のパラメータに入力すると、RAD_IOP から出力側のパラメータが返されます。

コールバックを設定していない場合は、ステータス フィールドを継続的にモニタして、このフィールドが RAD_IOP_STATUS_COMPLETE 以上になるのを待つことができます。次に、RAD_IOP_STATUS_DONE ステータスをクエリして、オープンが正常に行われたかどうかを判別します。


グループ: ソニー PS2 用 RAD_IOP API
関連する関数: RAD_IOPQueueAsyncOpenFile, RAD_IOP_OPENCALLBACK
関連する基本データ型: CHAR_PTR, RAD_IOP_STATUS_VALUES, S32, U32
