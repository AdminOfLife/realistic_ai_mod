Bink SDK 1.9c

SOUND_CORE_FLAGS

基本データ型 SOUND_CORE_FLAGS は、RAD_IOP が使用するサウンド フラグを指定します。

基本的な値

 値  説明
 RAD_IOP_NEED_CORE0 RAD_IOP に、コア 0 で PCM チャンネルを使用することを通知します。
 RAD_IOP_NEED_CORE1 RAD_IOP に、コア 1 で PCM チャンネルを使用することを通知します。
 RAD_IOP_SPDIF RAW ビットストリーム オーディオ データ (PCM 値ではなく) を S/PDIF ポートに出力するように RAD_IOP に指示します。
 RAD_IOP_NO_INIT_LIBSD RAD_IOP に、IOP 上の libsb を初期化しないように通知します (すでに初期化を完了しているため)。

説明

これらのフラグは RAD_IOPStartUp と RAD_IOPMemoryAmount で使用され、使用するサウンド リソースを指定します。必要な PCM サウンド チャンネルは 1 つだけですが、他の 1 つを直接使用する場合は 2 つとも要求できます。RAD_IOP では、送信サウンドをミックスすることもできます。したがって、Bink または Miles の外のオーディオを同時に再生する場合でも、実際には 2 つのサウンド コアは不要です。

PS2 の S/PDIF ポートは、sceSdSetCoreAttr 関数によって明示的にオフにしないかぎり、常にアクティブです。通常、S/PDIF ポートは、ライン出力端子に 48 kHz、16 ビット ステレオ PCM でオーディオ出力の「コピー」を出力します。ただし、RAD_IOP_SPDIF フラグを指定すると、サウンド コールバックに対して EE から返されるオーディオ データは、PCM データではなく、RAW ビットストリームとして処理されます。これは、DTS またはドルビー デジタル内蔵レシーバのために、S/PDIF ポートを通して 1.536 Mbit/s で送信されます。したがって、Miles Sound System の DTS エンコーダは、起動時に RAD_IOP_SPDIF フラグが指定されている必要があります (ビットストリーム送信にはコア 0 機能が必要なので、必ず OR 演算で RAD_IOP_NEED_CORE0 フラグと組み合わせる必要があります)。
