Bink のオーディオ トラック

説明

Bink では、オーディオ トラックをいくつでも Bink ファイルに追加できます。任意のトラックを解凍することも、解凍しないこともできます。たとえば、すべての音楽を 1 つのトラックに保存して、複数の言語を他のトラックに保存することもできます。

Bink の各オーディオ トラックは、モノラルかステレオのどちらかです。Bink ファイルを作成するときに、ミックスする各オーディオ ファイルに新しいトラック ID (デフォルトはトラック 0) を指定します。

通常は、ユーザーのサウンド ライブラリをオーディオの場所として Bink が自動的に指定します。再生するトラックを BinkSetSoundTrack 関数で指定するだけで、その後はすべて Bink が処理します。

Bink の高レベルのオーディオの関数 (BinkSetSoundTrack、BinkSetVolume、BinkSetPan) は、常にトラック ID を使用します。トラック ID は、オーディオをミックスするユーザーが RAD Video Tools で使用するときに使用する番号と同じです。

そのため、必要な操作はいくつかの標準を設定する (トラック 0 は音楽、トラック 1 は英語、トラック 2 はフランス語、など) ことだけです。これで、ビルドするロケールを指定する簡単な定義によってゲームは開始時の正しいトラックを選択し、制御することができます。

また、同じマルチトラック機能を使用して、5.1 サウンド トラックをサポートすることもできます。それには、個々のトラックに異なるチャンネルのペアを配置するだけです。詳細については、Xbox、Xbox 360、プレイステーション 3、および Miles Sound System のサウンドの説明を参照してください。

Bink には、低レベルのオーディオ関数もあります。この関数により、存在するオーディオ トラックを調べ、PCM サンプルを直接抽出できます。これらの関数は、トラック ID ではなくトラック インデックスを使用します。

トラック インデックスの範囲は、0 から、Bink ファイルにあるトラックの総数までになります。したがって、トラック ID が 42 と 118、65 と 12 などの場合、トラック インデックスの範囲は 0 から 1 になります。Bink ファイルにあるオーディオ トラックの総数は、BINK 構造体の NumTracks フィールドを見るとわかります。BinkGetTrackID 関数を使用してトラック インデックスとトラック ID を関連付けることができます。

しかし、ここで説明したように、原則としてオーディオの詳細は Bink が処理し、ユーザーは再生するトラックを選択するだけです。
