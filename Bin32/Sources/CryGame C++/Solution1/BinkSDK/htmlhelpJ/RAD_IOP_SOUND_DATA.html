RAD_IOP_SOUND_DATA

構造体 RAD_IOP_SOUND_DATA は、サウンド インタラプト コールバックの制御に使用する構造体です。

struct RAD_IOP_SOUND_DATA
{
  U32 which_sound_core;
  RAD_IOP_SOUNDCALLBACK callback;
  S32 overwrite;
  VOID_PTR address;
  U32 length;
  RAD_IOP_STATUS_VALUES status;
};

入力メンバ

which_sound_core
どちらのサウンド コアを登録するかを指定します (このコアは RAD_IOPStartUp 呼び出しで要求してあることが必要です)。
callback
サウンド コールバック関数のアドレスを指定します。

出力メンバ

overwrite
サウンド アドレス データを上書きするか (0 以外の値)、サウンド データをバッファにミックスするか (0) を示すフラグ。
address
コールバックがサウンドを流し込む先のアドレスを示します。
length
各コールバックに流し込むバイト単位の長さを示します。
status
サウンド コールバックの現在のステータスを示します。

説明

この構造体は、サウンド インタラプト コールバックを呼び出すデータを示します。これは、サウンド コールバックの登録が解除されるまで有効にしておく必要があります。

サウンド コールバックが実行されるたびに、sound_data 構造体の address および length フィールドで指定されたバッファにサウンド データを上書きするか、ミックスする必要があります。sound_data 構造体の overwrite フラグは、データを直接バッファにコピーする必要があるか (overwrite が TRUE)、またはデータを直接バッファにミックスする必要があるか (overwrite が FALSE) を示します。ミックスするには、S16 値をすでにバッファにある S16 値に加算します (-32768 〜 32767 に制限されます)。

通常、バッファは 48,000 Hz、16 ビットのステレオ PCM データ ブロックで、256 の左側サンプルの後ろに 256 の右側サンプルが続くソニー形式です。長さは常に 512 サンプルの倍数です (現在のバージョンの RAD_IOP では、2,048 バイト)。RAD_IOP_SPDIF + RAD_IOP_NEED_CORE0 フラグが使用されている場合、EE から返されるデータ ブロックは、RAW ビットストリーム データとして扱われ、DTS レシーバに送られます。RAD_IOP_SPDIF モードでは、overwrite フラグに意味はありません。
