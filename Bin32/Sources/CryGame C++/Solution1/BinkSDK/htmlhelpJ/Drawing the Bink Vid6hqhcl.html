ニンテンドー Wii での Bink ビデオ フレームの描画

説明

PC や Mac と比較して、ニンテンドー Wii でのビデオの再生ははるかに簡単です。ハードウェアがきわめて高速で、何よりも安定性があることが理由です。手元のマシンで問題なく再生できれば、他のマシンでも同様に再生できます。

Wii で画面にビデオ フレームをブリットする場合、バック バッファへコピーする方法と、マルチパス テクスチャを使用してレンダリングする方法があります。

バック バッファにコピーするほうが簡単であることは間違いありません。Bink で YUV データを直接画面の YUV のバック バッファにコピーするだけです。これは同時に、最大のパフォーマンスが得られる方法でもあります。レンダリングをループの外に置くことができるだけでなく、Bink 内で使用される YUV フォーマットから変換する必要がないためです。きわめて高速で、とても簡単です。この種類の再生のサンプルについては、ExamWiiBB.C を参照してください。

より柔軟なオプションは、GPU に指示して Bink 内部バッファから画面に直接レンダリングさせることです。この方法では、アルファ透過、アニメーション テクスチャ、またシーンに 3D オブジェクトを含むウィンドウ内のビデオなど、あらゆる種類の高度な操作を実行できます。

このように非スウィズル YUV データを直接レンダリングすることはかなりトリッキーで、調整にはかなりのテクスチャ ステージが必要です。幸い、このような作業をすべてラップする API を用意しました。これは、他のピクセル シェーダ/レジスタ コンバイナ プラットフォームとも互換性があるクロス プラットフォームの API です。

この種類の再生のサンプルについては、ExamWiiTex.C を参照してください。
