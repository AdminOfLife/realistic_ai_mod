ソニー PSP のメモリ管理

説明

デフォルトでは、Bink による動的メモリ管理は、C の標準関数 malloc と free を直接呼び出すことによって行います。もちろん、Bink の割り当て機能や解放機能を BinkSetMemory 関数で置き換えることもできます。

BinkGetSummary 関数を使用すると、動的メモリの使用量や最大使用量を確認できます。

リンク時のメモリ使用量を最小にするために、長時間をかけて Bink を PSP 上で微調整しました。Bink は、約 95K のコードのほか、アプリケーションとリンクした後は 25K のデータを必要とします。使用する Linked-in メモリを最小限に抑えるために、不要コードの削除機能を使用する必要があります。
