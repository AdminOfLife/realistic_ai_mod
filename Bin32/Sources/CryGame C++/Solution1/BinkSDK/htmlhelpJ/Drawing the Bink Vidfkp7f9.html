ソニー PS3 での Bink ビデオ フレームの描画

説明

ソニー PS3 での再生は PC より簡単です。ハードウェアが高速で、何よりも安定性があることが理由です。手元の開発キットで問題なく再生できれば、他の開発キットでも同様に再生できます。

ソニー PS3 の場合、ピクセル シェーダを使用して YUV を RGB 色空間に変換します。それには、Bink で OpenGL 8 ビット輝度テクスチャ (Y、cR、cB に 1 つずつの 3 つのうちの 2 つのセット) に直接解凍します。その後、BinkDoFrame を呼び出してからテクスチャをレンダリングするだけです。これには、BinkGetFrameBuffersInfo 関数と BinkRegisterFrameBuffers 関数を使用する必要があります。PSGLTextures.CPP ファイルには、シェーダとレンダリングに必要なすべてのコードが用意されています。また、ExamPS3.CPP ファイルには、それらの描画方法が示されています。

Windows、ソニー PS3、Wii、ゲームキューブ、Xbox、および Xbox 360 でピクセル シェーダ/レジスタ コンバイナ ブリッティングを実行するためのクロス プラットフォーム API が用意されています。詳細については、BinkTextures.h ファイルを参照してください。

アルファ平面を使用する場合の対処法もあります。アルファ平面を読み取ってブレンドするピクセル シェーダが用意されています。アルファ ビデオの表示方法を確認するには、ExamPS3A.CPP を参照してください。

PS3 の GPU は優れています。CPU に色変換の負荷をかけずに、HDTV ビデオを表示できます。
