Bink SDK 1.9c

BinkCopyToBufferRect

BinkCopyToBufferRect 関数は、Bink 内のメモリ バッファから指定されたメモリ アドレスにビデオのフレームの一部をコピーします。

S32 BinkCopyToBufferRect(
  HBINK bink,
  VOID_PTR dest_addr,
  S32 dest_pitch,
  U32 dest_height,
  U32 dest_x,
  U32 dest_y,
  U32 src_x,
  U32 src_y,
  U32 src_w,
  U32 src_h,
  BINK_COPY_FLAGS copy_flags
);

入力パラメータ

bink
Bink ハンドルを指定します。
dest_addr
コピー先のメモリ バッファのアドレス。
dest_pitch
コピー先バッファのピッチのバイト数。
dest_height
コピー先バッファの高さのピクセル数。
dest_x
X コピー先バッファへの (左) 方向のピクセル数で表されるオフセット。
dest_y
コピー先バッファへの X Y (上) 方向のピクセル数で表されるオフセット。
src_x
Bink 内バッファでの X 方向のピクセル数で表されるオフセット。BINKCOPYALL フラグを使用しない場合、この値は 16 の倍数である必要があります。
src_y
Bink 内バッファでの Y 方向のピクセル数で表されるオフセット。
src_w
コピーする幅のピクセル数。
src_h
コピーする高さのピクセル数。
copy_flags
コピー先バッファのタイプやその他のコピー オプションを表すフラグを指定します。

戻り値

フレームがスキップされた場合は 1、スキップされなかった場合は 0 が返されます。

説明

この関数を使用して、BinkCopyToBuffer と同様に、解凍されたピクセルを Bink 内の解凍バッファからコピー先のアドレスにコピーします。

この関数と BinkCopyToBuffer の唯一の違いは、この関数は Bink 内バッファの部分的な長方形領域もコピーできることです。ビデオのフレーム全体をコピーする必要がない場合や、1 つのビデオのフレームが複数の出力バッファに分割される (たとえば、タイル化された 256x256 のテクスチャを使用する) 場合に、この関数は便利です。

BINKCOPYALL を使用しないで、この関数を使用する場合は、パラメータ src_x は 16 の倍数である必要があります。
