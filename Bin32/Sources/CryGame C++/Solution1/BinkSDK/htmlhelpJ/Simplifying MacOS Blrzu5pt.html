MacOS でのブリッティングを BinkBuffers によって簡素化する

説明

では、どの方法を使用すればよいのでしょうか。画面に直接出力する方法でしょうか、それとも Gworlds を使用する方法でしょうか。この選択で迷った場合のために、Bink には BinkBuffer API があります。BinkBuffer によって MacOS の特殊性が隠され、これらの間の切り替えが簡単になります。MacOS 用の Bink プレイヤーのサンプルでは、BinkBuffer API を使用して Bink のサンプル画像を表示します。

ただし、BinkBuffer API は上位レベルの API であるため、すでにアプリケーションで問題なくブリッティングを実行する方法がわかっている場合は、BinkBuffers をまったく使用せずに、単に Bink でリニア バッファ (GWorld またはユーザー独自のフォーマットでは画面にすることもできます) に直接コピーする方法をお勧めします。

また、3D システムがアクティブな場合は BinkBuffer API を使用しないように注意してください。BinkBuffer API は比較的新しい 3D カードでは作動しますが、古いカード (または古いドライバ) では 2D データを 3D ウィンドウにブリットできないため、画面が表示されません。
