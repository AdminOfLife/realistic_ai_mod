ニンテンドー DS での Bink ビデオ再生

説明

ニンテンドー DS では、CPU を OS と共有する必要はありません。このため、一般的なニンテンドー DS の Bink 再生ループはとても単純です。Bink は BinkWait 関数でタイミングを処理するので、必要な処理は、ビデオの再生中にこの関数を確認することだけです。

ニンテンドー DS での再生の簡単なサンプルは次のようになります。

for ( ; ; )
{
  // フレームを解凍 
  BinkDoFrame( bink );


  // 必要に応じてフレームをスキップ...
  while ( BinkShouldSkip( bink )
  {
    BinkNextFrame( bink );
    BinkDoFrame( Bink );
  }


   // 前のフレームが表示されるまで待機
   wait_for_DMA_to_complete();


   // RGB16 にコピー
   BinkCopyToBuffer( bink, ... );


   // ビデオ フレームのアップロードを開始
   start_DMAing_the_video_frame();


  // 入力の確認、他のタスクの処理などを実行
  while ( BinkWait( bink ) )
    do_idle();


  // 次の vblank でバッファをフリップ
  schedule_flip();


  // ビデオが終了したか
  if ( bink->FrameNum == bink->Frames )
    break;

               
  // 次のフレームに移動
  BinkNextFrame( bink );  
}
