ソニー PSP での Bink ビデオ フレームの描画

説明

ソニー PSP でのビデオ再生は少し複雑ですが、非常に複雑なわけではありません。PSP はかなり低速なので (PS2 の半分の速度)、ソフトウェアで色変換を実行できません。残念ながら、PSP にはレジスタ コンバイナやピクセル シェーダがないので、GPU を使用する別の方法を見つける必要がありました。

Jason Dorie は、パレットをマルチパス ブレンディングと組み合わせて使用して、色変換を行う方法を考案しました。これはたいへん高速で、画質もよく、最小限のメモリしか使用しないため、素晴らしいアイデアです。ほとんど本物のピクセル シェーダと変わりません。

簡単にするために、Bink はすべての作業を行う小さな API を提供しています。テクスチャを割り当てる呼び出しを一度行い、それらを描画する呼び出しをもう一度行うだけです。実に単純で、Bink のピクセル シェーディング API とほとんど同じなので、移植も簡単です。また、アルファ平面を直接サポートするため、余分な開発が必要ありません。

唯一の複雑な点は、PSP での再生中にアイドル状態になれないことです。つまり、ビデオ フレームのレンダリング中に、次のビデオ フレームを解凍している必要があります。ほとんどのプラットフォームでは、これがデフォルトで行われますが、PSP では、ほとんどのソース コード サンプルが間違った動作を行います。次のフレームに進む前に、フレームがレンダリングされるまで待つ必要があります。

これはゲームに不都合なだけでなく、低速なマシンでのビデオ再生には致命的です。ほとんどのゲームでは、この問題を避ける独自のページ フリップが作成されているはずですが、作成していない場合は、ページ フリップを追加する簡単なスタンドアロン API を Bink が提供しています。これでゲームがはるかに高速になります。詳細については、pageflip.h ファイルを参照してください。

PSP の上手な再生の完全なサンプルについては、ExamPSP.CPP ファイルを参照してください。
