Bink SDK 1.9c

BINKBUFFER

BINKBUFFER 構造体は、BinkBufferOpen から返される構造体です。

struct BINKBUFFER
{
  U32 Width;
  U32 Height;
  VOID_PTR Buffer;
  S32 BufferPitch;
  BINK_COPY_FLAGS SurfaceType;
  U32 WindowWidth;
  U32 WindowHeight;
  U32 ScreenWidth;
  U32 ScreenHeight;
  U32 ScreenDepth;
};

出力メンバ

Width
BinkBuffer の幅を指定します。
Height
BinkBuffer の高さを指定します。
Buffer
内部ピクセル バッファのアドレスを指定します。この値は、BinkBufferLock 関数が呼び出されてから BinkBufferUnlock 関数が呼び出されるまでの間だけ有効です。
BufferPitch
内部ピクセル バッファのピッチをバイト単位で指定します。この値は、BinkBufferLock 関数が呼び出されてから BinkBufferUnlock 関数が呼び出されるまでの間だけ有効です。
SurfaceType
この BinkBuffer に適切にコピーするために BinkCopyToBuffer に渡す必要があるフラグを指定します。
WindowWidth
ウィンドウに対して設定する幅を指定します。これは、BinkBufferOpen に渡したウィンドウの位置とウィンドウ スタイルが考慮されています。
WindowHeight
ウィンドウに対して設定する高さを指定します。これは、BinkBufferOpen に渡したウィンドウの位置とウィンドウ スタイルが考慮されています。
ScreenWidth
現在の画面の幅をピクセル単位で指定します。これは、解像度の変更要求が正常に処理されたかどうかを確認するために使用できます。
ScreenHeight
現在の画面の高さをピクセル単位で指定します。これは、解像度の変更要求が正常に処理されたかどうかを確認するために使用できます。
ScreenDepth
現在の画面の色深度をビットで指定します (8、16、24、または 32)。これは、解像度の変更要求が正常に処理されたかどうかを確認するために使用できます。

説明

BINKBUFFER 構造体は、BinkBufferOpen が返す HBINKBUFFER ハンドルによってポイントされます。ポインタを逆参照することにより、これらの情報フィールドにアクセスできます。

これらの値は、決して変更しないように注意してください。
