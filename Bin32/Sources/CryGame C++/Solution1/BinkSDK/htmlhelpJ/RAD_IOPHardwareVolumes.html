Bink SDK 1.9c

RAD_IOPHardwareVolumes

関数 RAD_IOPHardwareVolumes は、サウンド コア ハードウェアのボリュームを設定します。

S32 RAD_IOPHardwareVolumes(
  U32 which_core,
  S32 master_core_left,
  S32 master_core_right,
  S32 pcm_channel_left,
  S32 pcm_channel_right
);

入力パラメータ

which_core
0 または 1 のどちらのサウンド コアを制御するかを指定します。
master_core_left
コア マスタの左のスピーカー出力について、0 から 0x3fff までの線形のボリュームを指定します。このボリューム レベルを設定しない場合は、-1 を渡します。
master_core_right
コア マスタの右のスピーカー出力について、0 から 0x3fff までの線形のボリュームを指定します。このボリューム レベルを設定しない場合は、-1 を渡します。
pcm_channel_left
PCM チャンネルの左のスピーカー出力について、0 から 0x7fff までの線形のボリュームを指定します。このボリューム レベルを設定しない場合は、-1 を渡します。
pcm_channel_right
PCM チャンネルの右のスピーカー出力について、0 から 0x7fff までの線形のボリュームを指定します。このボリューム レベルを設定しない場合は、-1 を渡します。

戻り値

正常ならば 0 以外の値、失敗の場合は 0 を返します。

説明

この関数は、ハードウェアのサウンド ボリュームを設定します。まだマスタ ボリュームを設定していない場合は、この関数を少なくとも 1 回呼び出してレベルを確立する必要があります。

マスタ ボリュームは、このチップ上のすべての音声のボリューム (ADPCM 音声を含む) を制御します。PCM チャンネルのボリュームでは、PCM チャンネルだけが設定されます。すべての Bink および Miles 音声が同じ IOP 音声を使用するため、この関数を使用して基準を設定してから、通常の Bink または Miles 関数呼び出しを使用して、ソースごとにボリュームを設定します。

ただし、サウンド コア 0 はサウンド コア 1 に入力されるので、コア 0 を使用する場合はコア 1 のマスタ ボリュームも調整する必要があります。調整しないと何も聞こえません。
