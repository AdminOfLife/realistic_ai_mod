GNU/Linux での Bink 再生

説明

GNU/Linux では、他のアプリケーションと CPU 時間を共有することをお勧めしますが、必須ではありません。CPU 時間を共有する場合は、各フレームを待っている間にイベントを処理する必要があります。Bink は BinkWait 関数でタイミングを処理するので、必要な処理は、イベントを処理していないときは常にこの関数を確認するということだけです。

そのため、イベント ループの BinkWait 関数のモニタだけを行います。次のフレームに移動する時間になったことを BinkWait が通知したら、そのままフレームを進めるコードを呼び出します。

SDL ライブラリを使用した GNU/Linux での再生の簡単なサンプルを以下に示します。

while ( !done )
{
  // Linux イベントのチェック
  while ( SDL_PollEvent( &event ) )
  {
    if ( event.type == SDL_QUIT )  
      done = 1;
    else
    {
      // イベントを処理するコードの残りの部分
    }
  }
  else
  {
    // イベントがないので再生をチェック
    if ( !BinkWait( bink ) )
    {
      // 次の Bink フレームの処理に進む
      Show_next_frame( bink, screen, bink_surface_type );
    }
  }
}
