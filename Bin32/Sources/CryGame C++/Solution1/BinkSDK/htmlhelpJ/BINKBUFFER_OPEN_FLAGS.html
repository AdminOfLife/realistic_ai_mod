BINKBUFFER_OPEN_FLAGS

基本データ型 BINKBUFFER_OPEN_FLAGS は、開こうとする BinkBuffer のタイプと機能を表します。

基本的な値

 値  説明
 BINKBUFFERAUTO 使用できる最善の BinkBuffer ブリッティング スタイルを選択します。Win32 では、Bink は BINKBUFFERPRIMARY から BINKBUFFERRGBOFFSCREENSYSTEM までのすべてのブリッティング スタイルを開いて、解像度と拡大縮小要件に一致する最善のブリッティング スタイルを探します。MacOS では、Bink は BINKBUFFERDIRECT と BINKBUFFERGWORLD から、仕様に一致する最善のスタイルを探します。
 BINKBUFFERPRIMARY Win32: BinkBuffer をプライマリ サーフェスに対して開きます。このブリッティング スタイルは、組み込みの Bink YUV から RGB への変換機能を使用するため、表示に問題が発生しないことが保証されており、ハードウェア変換スタイルと比較して、わずかに遅く表示される場合があるだけです。プライマリ サーフェス ブリッティング スタイルは、拡大縮小とクリッピングを一切実行できません (ただし BinkCopyToBuffer のパラメータに 2 倍拡大のフラグを指定して、単純に 2 倍に拡大することはできます。このブリッティング スタイルでは、ピクセルが直接 BinkBufferLock/BinkBufferUnlock ブロック内部のオンスクリーンに置かれるため、BinkBufferBlit が実行する必要がある処理はまったくありません)。
 BINKBUFFERDIBSECTION Win32: Windows DIBSection の最上部のシン レイヤ BinkBuffer を開きます。これは、DirectDraw を必要としない唯一のブリッティング スタイルです。BinkBuffer DIB は、ブリッティングをできるだけ高速にするために、デスクトップと同じフォーマットで作成されます。このブリット タイプでも、BINKBUFFERRESOLUTION フラグを使用できます。この場合、DirectDraw でなく ChangeDisplaySettings を使用してモードが切り替えられます。
 BINKBUFFERYV12OVERLAY Win32: BinkBuffer を YV12 オーバーレイ サーフェスに対して開きます。これは最速のブリッティング スタイルですが、サポートするビデオ カードはほとんどありません (ATI と nVidia の一部のカードがサポートしています)。このブリッティング スタイルでは、拡大縮小とクリッピングが可能な場合とできない場合があります。BinkBufferBlit は、このブリッティング スタイルでピクセルを画面にコピーするのではなく、カラーキー ピクセルを画面にブリットします (これによってオーバーレイ ピクセルが透過して表示されます)。カラーキー ピクセルがすでに画面に存在している場合、変換されたピクセルは、プライマリ サーフェスのブリッティング スタイルと同様に、BinkBufferLock/BinkBufferUnlock ブロックに書き込まれるとすぐに表示されます。
 BINKBUFFERUYVYOVERLAY 次を参照。
 BINKBUFFERYUY2OVERLAY Win32: この 2 つのフラグは、BinkBuffer を YUY2 または UYVY オーバーレイ サーフェス上に開きます。これらは最も一般的なオーバーレイ サーフェスで、ほとんどすべてのビデオ カードでサポートされています。このブリッティング スタイルでは、拡大縮小とクリッピングが可能な場合とできない場合があります。BinkBufferBlit は、このブリッティング スタイルでピクセルを画面にコピーするのではなく、カラーキー ピクセルを画面にブリットします (これによってオーバーレイ ピクセルが透過して表示されます)。カラーキー ピクセルがすでに画面に存在している場合、変換されたピクセルは、プライマリ サーフェスのブリッティング スタイルと同様に、BinkBufferLock/BinkBufferUnlock ブロックに書き込まれるとすぐに表示されます。
 BINKBUFFERYV12OFFSCREEN Win32: BinkBuffer を YV12 オフスクリーン バッファに対して開きます。このブリッティング スタイルでは、ビデオ ドライバはブリッティング処理の中で YUV データを RGB に変換します。このブリッティング スタイルは、拡大縮小とクリッピングが可能な場合とできない場合があります。BinkBufferBlit は、ピクセルを画面にコピーします。BinkBufferLock/BinkBufferUnlock ブロックは、Bink ピクセルをオフスクリーン バッファにロードします。Bink のソフトウェア ブリッタは、通常このブリッティング スタイルより高速です。
 BINKBUFFERUYVYOFFSCREEN 次を参照。
 BINKBUFFERYUY2OFFSCREEN Win32: この 2 つのフラグは、BinkBuffer を YUY2 または UYVY オフスクリーン バッファ上に開きます。このブリッティング スタイルでは、ビデオ ドライバはブリッティング処理の中で YUV データを RGB に変換します。このブリッティング スタイルは、拡大縮小とクリッピングが可能な場合とできない場合があります。BinkBufferBlit は、ピクセルを画面にコピーします。BinkBufferLock/BinkBufferUnlock ブロックは、Bink ピクセルをオフスクリーン バッファにロードします。ほぼすべてのビデオ カードは、少なくともこのレベルのハードウェア YUV から RGB への変換をサポートしています。Bink のソフトウェア ブリッタは、通常このブリッティング スタイルより高速です。
 BINKBUFFERRGBOFFSCREENVIDEO Win32: BinkBuffer をビデオ カード RAM 上のオフスクリーン RGB バッファに対して開きます。このブリッティング スタイルでは、Bink は YUV データをオフスクリーン RGB フォーマットに変換します。このブリッティング スタイルは、ほとんどの場合、拡大縮小やクリッピングが可能です。BinkBufferBlit は、ピクセルを画面にコピーします。BinkBufferLock/BinkBufferUnlock ブロックは、RGB Bink ピクセルをオフスクリーン バッファに取り込みます。
 BINKBUFFERRGBOFFSCREENSYSTEM Win32: BinkBuffer をシステム カード RAM 上のオフスクリーン RGB バッファに対して開きます。このブリッティング スタイルでは、Bink は YUV データをシステム メモリ内のオフスクリーン RGB フォーマットに変換します。このブリッティング スタイルは、拡大縮小とクリッピングをサポートしますが、すべてのブリッティング スタイルの中で最も低速です。BinkBufferBlit は、ピクセルを画面にコピーします。BinkBufferLock/BinkBufferUnlock ブロックは、RGB Bink ピクセルをオフスクリーン バッファに取り込みます。原則として、このオプションではなく BINKBUFFERDIBSECTION スタイルを使用してください。このスタイルは、同等以上の機能を持ち、幅広い互換性も備えています。
 BINKBUFFERDIRECT MacOS: BinkBuffer を direct-to-screen (画面への直接出力) モードで開きます。このブリッティング スタイルは、組み込みの Bink YUV から RGB への変換機能を使用するため、表示に問題が発生しないことが保証されており、動作もきわめて高速です。direct-to-screen ブリッティング スタイルは、拡大縮小とクリッピングを一切実行できません (ただし BinkCopyToBuffer のパラメータに 2 倍拡大のフラグを指定して、単純に 2 倍に拡大することはできます。このブリッティング スタイルでは、ピクセルが直接 BinkBufferLock/BinkBufferUnlock ブロック内部のオンスクリーンに置かれるため、BinkBufferBlit が実行する必要がある処理はまったくありません)。
 BINKBUFFERGWORLD MacOS: BinkBuffer を標準の MacOS GWorld グラフィック バッファに対して開きます。このブリッティング スタイルは、MacOS マシンでは最も互換性がありますが、BINKBUFFERDIRECT より遅くなります。このブリッティング スタイルは、クリッピングも拡大縮小も可能です。このブリッティング スタイルでは、BinkCopyToBuffer は GWorld にコピーするので、BinkBufferBlit を使用してピクセルを画面にブリットします。
 BINKBUFFERRESOLUTION このフラグを前述のいずれかのブリッティング スタイルに対して指定すると、BinkBuffer が開いたときに Bink はビデオ モードの解像度を切り替えます。このフラグを使用する場合は、あらかじめ BinkBufferSetResolution 関数を呼び出して、切り替える解像度を Bink に通知する必要があります。要求されたモードに Bink が切り替えることができない場合でも、BinkBuffer は開きます。BinkBuffer 構造体の ScreenWidth、ScreenHeight、および ScreenDepth フィールドを調べて、要求したモードが正常に設定されたかどうかを確認する必要があります。
 BINKBUFFERSTRETCHXINT 次を参照。
 BINKBUFFERSTRETCHX 次を参照。
 BINKBUFFERSHRINKXINT 次を参照。
 BINKBUFFERSHRINKX これらのフラグの少なくとも 1 つを使用して、実行する X (幅) 方向の拡大や縮小のタイプを指定します。名前の末尾が「INT」であるフラグは、整数倍の拡大または縮小 (2 倍に拡大、または半分に縮小など) を意味します。多くのカードでは、整数倍の拡大縮小だけを実行できます。したがって、整数倍の拡大縮小だけが必要な場合は必ず「INT」フラグを指定してください。指定しないと、BinkBuffer を開けないことがあります。
 BINKBUFFERSTRETCHYINT 次を参照。
 BINKBUFFERSTRETCHY 次を参照。
 BINKBUFFERSHRINKYINT 次を参照。
 BINKBUFFERSHRINKY これらのフラグの少なくとも 1 つを使用して、実行する Y (高さ) 方向の拡大や縮小のタイプを指定します。名前の末尾が「INT」であるフラグは、整数倍の拡大または縮小 (2 倍に拡大、または半分に縮小など) を意味します。多くのカードでは、整数倍の拡大縮小だけを実行できます。したがって、整数倍の拡大縮小だけが必要な場合は必ず「INT」フラグを指定してください。指定しないと、BinkBuffer を開けないことがあります。

説明

これらのフラグは、BinkBufferOpen 関数の open_flags パラメータとして使用します。これらのフラグにより、必要な YUV ブリッティング バッファのタイプと機能を指定します。

各種の拡大縮小フラグを指定しない場合でも、BinkBufferSetScale 関数で拡大縮小ができるブリッティング スタイルを取得できる場合がありますが、原則として取得できません。したがって、拡大縮小することがわかっている場合は、BinkBufferOpen の呼び出し時に常にいずれかの拡大縮小フラグを使用する必要があります。
