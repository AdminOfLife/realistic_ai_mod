ソニー PS2 でのファイル I/O

説明

ソニー PS2 で非同期 I/O をサポートするために、Bink は IOP チップとのインターフェイスを持つ (IOP チップは I/O を実行) 高度で使いやすい API を提供しています。この API については、「ソニー PS2 用の RAD_IOP API」に説明されています。

RAD_IOP API は、CD または DVD からの読み取りには sceCdRead を使用し、ホストまたはハード ドライブからの読み取りには C ランタイム関数 (open、lseek、read と close) を使用します。この非同期 API を使用して、Bink のムービー再生と並行してディスクを読み取ることができます。この API は複数のファイルから一度にストリーミングできるように作成されています。

ファイル名は、標準の PS2 名形式であることが必要です。CD または DVD ファイルには、プリフィックス「cdrom0:」とポストフィックス「;1」が必要です。ホストまたはハード ドライブ ファイルの場合は、「host0:」または「hdd0:」プリフィックスを使用します (2.8 システム ライブラリを使用した host0: からの読み取りは、3.0 ライブラリよりかなり低速です)。

Bink は CD と DVD をストリーム モードで使用します。ノミナル モードは、ディスク上の位置によっては高速ですが、失敗することがあり、その場合は再読み取りが行われるため遅くなります。Bink は、ビデオをストリーミングしている間は、再読み取りが起きても停止できません。したがって、多少遅くてもストリーム モードを使用するほうが安全です (ソニーは、すべてのビデオ再生システムでこれを推奨しています)。CD や DVD の I/O はきわめて高速なので、ディスクの場所や読み取りモードに関係なく、推奨する PS2 データ レートの 500 Kps から 700 Kps に容易に対応します。

sceRead を呼び出す前には、必ず sceCdSync を呼び出しています。したがって、他の API がディスクにアクセスしていても (Multistream など)、過剰なアクセスでシーク タイムのために Bink I/O バッファが空になる (ムービーのスキップが起きます) ことがないかぎり、正常に実行されます。

BinkSetIO 関数を使用して Bink I/O システム全体を完全に上書きすることもできますが、お勧めしません。PS2 上で PS2 I/O システムを効率よく、また高い信頼性で実行することは非常に難しいことです。IOP ライブラリは、ゲームと容易に共存できるように作成されています。
