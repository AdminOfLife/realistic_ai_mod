RAD_IOPSetSPUVoiceLoopAddress

関数 RAD_IOPSetSPUVoiceLoopAddress は、1 つ以上の SPU 音声のループ バック先のアドレスを設定します。

S32 RAD_IOPSetSPUVoiceLoopAddress(
  U32 core0_voice_mask,
  U32 core1_voice_mask,
  U32 loop_address
);

入力パラメータ

core0_voice_mask
設定するコア 0 の音声のビット マスクを指定します。
core1_voice_mask
設定するコア 1 の音声のビット マスクを指定します。
loop_address
ループ バック先の SPU RAM 上のループ アドレスを指定します。

戻り値

正常ならば 0 以外の値、失敗の場合は 0 を返します。

説明

この関数は、再生の末尾に達したときにジャンプする位置を SPU に指示します。

ほとんどのサウンド システムとは異なり、ループするかどうかはソース ファイル内で SPU に指示します。つまり、サウンドをループするかどうかを VAGEdit 内で選択します。サウンドがループされる場合は、この関数を使用して、バッファの末尾に達したときの SPU RAM 上のジャンプ先を変更できます。

ループするかどうかを実行時に選択できると便利なので、そのためのトリッキーな方法があります。VAG ファイルを作成する場合は、必ずループ モードにします。これで、すべてのサウンド サンプルが永久にループするようになります。ただし、トリックとして、無音データを含む特別な SPU アドレスに戻るループ アドレスを設定します。これで、サウンドが一度再生されると、SPU 無音アドレスにジャンプし、その後は無音が再生され続けます。

このように、ループ アドレスを SPU 無音アドレスに設定するかどうかにより、サウンドをループするかどうかを選択できます。この手法のサンプルについては、MilesPS2.c サンプル プログラムを参照してください。思ったよりはるかに簡単です。RAD_IOP には、SPU 無音アドレスを参照する定数 SPU_SILENCE_ADDRESS が定義されています。
