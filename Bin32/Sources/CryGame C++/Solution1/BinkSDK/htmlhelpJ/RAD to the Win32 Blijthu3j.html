RAD による Win32 ブリッティングの解決

説明

では、どの技術を使用すればよいのでしょうか。Bink の組み込みソフトウェア、オーバーレイ、オフスクリーン バッファ、3D ブリッティングのどれでしょうか。

アプリケーションが 2D の場合は、DirectDraw の複雑さを隠す BinkBuffer API が用意されているため、さまざまな色空間オプションの切り替えは簡単です。Bink プレイヤー、サンプル ExamWin.C、サンプル ExamMSS.C は、いずれも BinkBuffer API を使用することによって簡単に Bink フレームを表示しています。

しかし、BinkBuffer API はかなり高レベルなので、すでに DirectDraw をアプリケーションで実行している場合は、BinkBuffer を完全にスキップし、内部の Bink ソフトウェア ルーチンを使用してプライマリ サーフェスまたはセカンダリ サーフェスに直接変換することをお勧めします。この種類の再生のサンプルについては、ExamDDP.C file を参照してください。

また、3D システムがアクティブな場合は BinkBuffer API を使用しないように注意してください。BinkBuffer API は一部の 3D カードでは作動しますが、古いカード (または古いドライバ) では 2D データを 3D ウィンドウにブリットできないため、画面が表示されません。

3D アプリケーションの場合は、3D ハードウェアの呼び出しを簡略化するいくつかの API が用意されています。ほとんどの場合は、サンプル コードを変更することさえ必要ありません。ピクセル シェーダ ブリッティングについては、BinkTextures.h API ファイルを参照してください。固定機能 3D ブリッティングについては、RAD3D.h API ファイルを参照してください。

これらの技術の 1 つを使用して、アプリケーション内でビデオをすばやく簡単に起動して実行してください。
