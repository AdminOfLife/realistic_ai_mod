RAD_IOPQueueAsyncOpenFile

関数 RAD_IOPQueueAsyncOpenFile は、ファイル ハンドルを非同期で開きます。

S32 RAD_IOPQueueAsyncOpenFile(
  RAD_IOP_ASYNC_OPEN_DATA *open_data
);

出力パラメータ

open_data
オープンの入力パラメータと出力パラメータを指定します。

戻り値

オープンが正常にキューに置かれると 0 以外の値、失敗すると 0 を返します。

説明

この関数は、ファイル オープンをキューに置き、バックグラウンドで実行されるようにします。name フィールドと、オプションで group フィールドおよび callback フィールドに入力してから、この関数を呼び出します。

name フィールドには、通常の PS2 フォーマットの名前を指定できます (「cdrom0:\\MOVIES\\CREDITS.BIK;1」または「host:c:\\movies\\credits.bik」)。

完了したことを確認するには、コールバックを使用するか、status フィールドを調べます。ステータスが RAD_IOP_STATUS_DONE に設定されていれば、file_handle および size フィールドが正しく設定されています。

RAD_IOPCloseFile を呼び出して、この関数で開いたすべてのハンドルを閉じる必要があります。

ファイル ハンドルの値は、次の 4 つの種類のいずれかです。
1) ホストまたは外部ハード ドライブ上のファイル。これは、IOP C ランタイムの open 関数から返される値です。
2) CD または DVD ファイルの場合は、基本的に開始セクタ番号です (RAD_IOPGetFileHandleToLSN を使用してエンコードされます)。
3) IOP メモリへのポインタ (RAD_IOPGetFileHandleToIOPMemory を使用してエンコードされます) - IOP RAM からのデータの取得に使用されます。
4) SPU メモリへのポインタ (RAD_IOPGetFileHandleToSPUMemory を使用してエンコードされます) - SPU RAM からのデータの取得に使用されます。

最初の種類のファイル ハンドルを取得するためには、この関数が必要です。また、この関数は、CD または DVD 上のファイル名の開始セクタを検索して、2 番目の種類を返すこともできます。ただし、ファイルの論理セクタ番号がすでにわかっている場合は、RAD_IOPGetFileHandleToLSN のほうが高速です。

open_data 構造体は、操作が終了するまで有効にしておく必要があります。したがって、ステータスが RAD_IOP_STATUS_DONE 以上になるまでメモリを解放しないでください。
