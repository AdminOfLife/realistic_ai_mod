Bink SDK 1.9c

BinkBufferSetOffset

関数 BinkBufferSetOffset は、デスクトップ上のウィンドウの位置またはウィンドウ内の再生オフセットが変更されたことを BinkBuffer に通知します。

S32 BinkBufferSetOffset(
  HBINKBUFFER bink_buf,
  S32 dest_x,
  S32 dest_y
);

入力パラメータ

bink_buf
BinkBuffer ハンドルを指定します。
dest_x
ブリットする X (左) 座標 (ウィンドウの左上角からの相対位置) を指定します。
dest_y
ブリットする Y (上) 座標 (ウィンドウの左上角からの相対位置) を指定します。

戻り値

BinkBuffer が正常に位置変更できたか (0 以外の値)、指定された座標が無効だったか (0) を返します。

説明

この関数は、再生ウィンドウの位置自体、またはウィンドウ内での再生オフセットが変更されたことを BinkBuffer に通知します。

通常、Win32 では、この関数をウィンドウ プロシージャの WM_WINDOWPOSCHANGED メッセージから呼び出します。dest_x と dest_y はクライアント座標のオフセットであるため、ビデオをウィンドウの左上角で再生する場合は、どちらのオフセットも 0 に設定します。したがって、WM_WINDOWPOSCHANGED メッセージのデフォルトのコードは次のようになります。

case WM_WINDOWPOSCHANGED:
  if ( bink_buf )
    BinkBufferSetOffset( bink_buf, 0, 0 );
  break;

ビデオ再生ウィンドウを新しい位置にドラッグしても、DirectDraw オーバーレイは移動しないため、この関数呼び出しは失敗します。

この関数は、BinkBuffer の位置を変更できれば 0 以外の値、BinkBuffer を指定した位置に移動できなければ 0 を返します。
