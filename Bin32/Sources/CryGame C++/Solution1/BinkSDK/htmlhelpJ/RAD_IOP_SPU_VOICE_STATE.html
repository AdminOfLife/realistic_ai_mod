Bink SDK 1.9c

RAD_IOP_SPU_VOICE_STATE

構造体 RAD_IOP_SPU_VOICE_STATE は、RAD_IOPQueueAsyncSPUVoiceState 関数で使用される構造体です。

struct RAD_IOP_SPU_VOICE_STATE
{
  U32 core;
  U32 voice;
  RAD_IOP_SPU_VOICE_STATE_CALLBACK callback;
  RAD_IOP_STATUS_VALUES status;
  S32 current_playing;
  U32 current_address;
  U32 current_left_volume;
  U32 current_right_volume;
  U32 current_envelope;
};

入力メンバ

core
クエリするコアを指定します (0 または 1)。
voice
クエリする音声を指定します (0 〜24)。一度に 1 つの音声だけをクエリするので、これは音声マスクではなく、音声のインデックスです。
callback
終了時に実行するコールバックを指定します。コールバックが不要な場合は、このフィールドに 0 を設定します。

出力メンバ

status
非同期ステータスの現在のステータスを示します。
current_playing
再生されている場合は 1、再生されていない (音声エンベロープがリリース段階にまだ入っていない) 場合は 0。
current_address
音声再生の現在のアドレスを返します。
current_left_volume
左の音量の値を返します。これは、現在のエンベロープ値によって調整された左の音量です。
current_right_volume
右の音量の値を返します。これは、現在のエンベロープ値によって調整された右の音量です。
current_envelope
現在のエンベロープ値を返します。

説明

この構造体は、RAD_IOPQueueAsyncSPUVoiceState 関数の入力と出力の両方で使用されます。入力パラメータを提供すると、RAD_IOP が出力パラメータを返します。

コールバックを設定していない場合は、ステータス フィールドを継続的にモニタして、このフィールドが RAD_IOP_STATUS_COMPLETE 以上になるのを待つことができます。次に、RAD_IOP_STATUS_DONE ステータスをクエリして、クエリが正常に行われたかどうかを判別します。


グループ: ソニー PS2 用 RAD_IOP SPU ハードウェア音声 API
関連する関数: RAD_IOPQueueAsyncSPUVoiceState, RAD_IOP_SPU_VOICE_STATE_CALLBACK
関連する基本データ型: RAD_IOP_STATUS_VALUES, S32, U32
