Bink SDK 1.9c

BinkDoFrameAsync

The function BinkDoFrameAsync decompresses the next frame of video on a background thread.

S32 BinkDoFrameAsync(
  HBINK bink,
  U32 yplane_thread_index,
  U32 other_thread_index
);

In Parameters

bink
Specifies the Bink handle.
yplane_thread_index
Specifies the thread index that the Y plane should be decompressed on.
other_thread_index
Specifies the thread index that the cR, cB, and alpha planes should be decompressed on (and the sound).

Returns

Returns one on success, zero if there was an error.

Discussion

This function does the actual decompression of the next Bink frame. This function runs in the background on Windows, the Xbox 360 and PS3. You can find out when the decompression is completed with the BinkDoFrameAsyncWait function.

Bink decompresses the compressed Bink frame into the three internal YUV memory buffers in this function (like BinkDoFrame) only in the background.

In Bink, most of the decompression time is spent decompressing the Y plane, so the work can be split between two threads - one for the Y plane decompression and one for "everything else" (the cR, cB and alpha plane and the audio decompression).

In general, though, you really only need one background thread for Bink decompression - the only time you really need two threads is if you are decompressing a lot of movies at once (or if your movie is extremely high-resolution - beyond even 1080p). So, normally, you will just pass the same thread number for both the Y and other thread indexes - all of the work will then be done on just one background thread.

The thread_index values must have been previously started with the BinkStartAsyncThread function.
