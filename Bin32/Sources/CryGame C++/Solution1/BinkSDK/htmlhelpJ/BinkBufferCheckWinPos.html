Bink SDK 1.9c

BinkBufferCheckWinPos

関数 BinkBufferCheckWinPos は、指定されたウィンドウの位置座標を現在の BinkBuffer タイプに適合するように調整します。

void BinkBufferCheckWinPos(
  HBINKBUFFER bink_buf,
  S32 *new_window_x,
  S32 *new_window_y
);

入力パラメータ

bink_buf
BinkBuffer ハンドルを指定します。

出力パラメータ

new_window_x
チェックする X (左) 座標を指定します。
new_window_y
チェックする Y (上) 座標を指定します。

説明

この関数は、提供されたウィンドウの位置座標が現在の BinkBuffer ブリッティング スタイルに適合するように調整します。各ブリッティング スタイルにはそれぞれ制約があるため (プライマリ サーフェス ブリッティング スタイルは部分的にオフスクリーンになるように配置できない、一部のオーバーレイ ハードウェアは部分的なオフスクリーン サーフェスを受け入れない、など)、この関数で再生ウィンドウを安全にドラッグします。

通常、Win32 では、この関数を WM_WINDOWPOSCHANGING メッセージ ハンドラで呼び出して、ウィンドウが無効な場所に決して移動しないようにする必要があります。NewWindowX と NewWindowY 座標はグローバル画面座標であるため、WM_WINDOWPOSCHANGING のデフォルトの処理ロジックは次のようになります。

case WM_WINDOWPOSCHANGING:
  if ( ( ( ( WINDOWPOS* ) lparam )->flags & SWP_NOMOVE ) == 0 )
  {
    if ( bink_buf )
    {
      S32 x,y;

      x = ( ( WINDOWPOS* ) lparam )->x;
      y = ( ( WINDOWPOS* ) lparam )->y;
      BinkBufferCheckWinPos( bink_buf, &x, &y );
      ( ( WINDOWPOS* )lparam )->x = x;
      ( ( WINDOWPOS* )lparam )->y = y;
    }
  }
  break;
