ソニー PS2 のサウンド

説明

ソニー PS2 でサウンドをサポートするために、Bink は IOP チップとのインターフェイスを持つ (IOP チップは SPU サウンド出力チップに通知) 高度で使いやすい API を提供しています。この API については、「ソニー PS2 用の RAD_IOP API」に説明されています。

RAD_IOP のセクションで詳細に説明されているように、基本的に Bink は PS2 上の各 SPU がサポートする 2 つの PCM 出力チャンネルの 1 つだけを必要とします。どちらのチャンネルを使用するかは、BinkSoundUseRAD_IOP 呼び出しで指定できます。

Bink では、すべてのオーディオをあらかじめ 1 つにミックスしてから、1 つの SPU チャンネルに渡します。たとえば、複数のサウンド トラックを持つ複数の Bink を再生できますが、その場合も使用するハードウェア サウンド チャンネルは 1 つだけです。このサウンド ミックスのステップでは、サウンドのフォーマット変換も行うので、Bink ファイル内のすべての周波数およびフォーマットのオーディオを使用できます。ここでは、PS2 ハードウェア フォーマットである 48,000 Hz、16 ビット、ステレオ フォーマットに自動的に変換します。

この PCM サウンド チャンネル以外は、他のサウンド機能はまったく使用していません。したがって、任意の API (Multistream など) を使用して、他の 48 ADPCM 音声や他の PCM チャンネルまで制御できます。使用することを宣言した同じハードウェア音声を激しく使用するものを除いて、Bink はすべてのサウンド出力ミドルウェアに対応する予定です。

RAD_IOP API を使用しない場合は、このオーディオ サポートをユーザー自身で作成することはまったく問題ありません。電子メールをいただけば、変更可能なサンプル コードをお送りします (しかし、頑強で他の障害にならないように作成されている RAD_IOP ライブラリの使用をお勧めします)。
