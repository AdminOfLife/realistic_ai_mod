Bink SDK 1.9c

BINK_GOTO_FLAGS

基本データ型 BINK_GOTO_FLAGS で、どのようにジャンプを実行するかを指定します。

基本的な値

 値  説明
 NULL デフォルト (フラグなし) では、Bink は最も近いキー フレームを検索し、指定されたフレームまで解凍してジャンプを実行します。
 BINKGOTOQUICK このフラグを使用すると、最初にキー フレームを検索せずに、Bink を強制的に特定のフレームに直接ジャンプさせることができます (ジャンプ後のフレームが正しく表示されないことがあります)。
 BINKGOTOQUICKSOUND Bink は、オーディオの最初の 750 ミリ秒をムービーの最初のフレームに保存します。つまり、フレーム 500 に移動する場合は、再生を再開してすぐにサウンドを開始するために、Bink の内部ではフレーム 500 よりも 750 ミリ秒前にジャンプする必要があります。

サウンドを目的のフレームから正確に開始するためのこの微調整は、実行のために追加の I/O が発生することと CPU がわずかながら消費されることを除くと、ユーザーから見えないところで処理されます。

BINKGOTOQUICKSOUND フラグを使用すると、Bink は 750 ミリ秒前に戻らないので I/O と CPU を節約できますが、BinkGoto の後の最初の 750 ms が無音になります。これは、できるだけ短時間でジャンプする必要があり、サウンドのことを考慮しない場合に便利です。

説明

これらのフラグを BinkGoto 関数で使用して、実行するジャンプの種類を指定します。

BinkGetKeyFrame 関数を使用して、最も近いキー フレームを手動で検索することもできます。
