Bink でのエラー発生時のファイル I/O 処理

説明

再生中に発生したエラーに対して Bink はきわめて保守的です。不正なデータを読み込んでクラッシュする危険を冒すのではなく、すぐに再生を終了します。

I/O エラーが発生すると、Bink は BINK 構造体の ReadError フィールドを 0 以外に設定し、後続のフレームをすべてスキップします。したがって、I/O エラーをチェックしない場合は再生ロジックは正常に終了し、ユーザーが特別なことをする必要はありません。

I/O エラーをチェックする (たとえば、メッセージを表示する) 場合は、BinkNextFrame を呼び出すたびに ReadError が 0 以外になっていないかだけをチェックします。

エラーの後でムービーの再生を再開するには、Bink ムービーを閉じて開き直してください。再生を中断した位置に戻るには、BinkGoto 関数を使用して正しい位置に移動します。
