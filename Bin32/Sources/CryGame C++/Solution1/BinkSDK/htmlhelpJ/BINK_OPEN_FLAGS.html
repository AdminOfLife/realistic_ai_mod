Bink SDK 1.9c

BINK_OPEN_FLAGS

基本データ型 BINK_OPEN_FLAGS は、Bink ファイルを開く特殊な方法を表します。

基本的な値

 値  説明
 BINKFILEHANDLE このフラグを使用して、指定した名前がファイル名ではなく、ファイル ハンドルであることを Bink に通知します。それにより、Bink は新しくファイルを開かずに、このハンドルを使用します。このフラグを使用する場合は、ファイル ハンドルを char* に型キャストする必要があります。このフラグは、複数の Bink ファイルを含むリソース ファイルを扱う場合に便利です。このフラグを使用した場合、Bink はファイル ハンドルを開きません (また、閉じません)。

Win32、Xbox、および Xbox 360 では、ファイル ハンドルの値は、システム関数 CreateFile が返すハンドルです。

ソニー PS3 では、ファイル ハンドルの値は、C ランタイム関数 open が返すポインタです。

ソニー PSP では、ファイル ハンドルの値は、sceIOOpen が返します。

ソニー PS2 では、ファイル ハンドルの値は、RAD_IOPQueueAsyncOpenFile が返す値か、論理セクタ番号から構成されるファイル ハンドル、または IOP C ランタイムのオープン関数が返すファイル ハンドルです。

MacOS では、ファイル ハンドルの値は、システム関数 FSpOpenDF が返すポインタです。

GNU/Linux では、ファイル ハンドルの値は、通常のシステム関数 fopen が返すハンドルです。

ニンテンドー ゲームキューブまたは Wii では、ファイル ハンドルの値は、システム関数 DVDFastOpen が返す DVDFileInfo ポインタです。ゲームキューブおよび Wii では、DVDFileInfo ポインタの userData フィールド (fileInfo->cb.userData) に、このファイル ハンドルに対応するビデオの開始ファイル オフセット (通常は 0) を入力する必要があります。このような特殊な処理が必要になる理由は、現在のファイルの位置を決定するシステム関数がないためです。この開始ファイル オフセットは、32 の倍数である必要があります。
 BINKFROMMEMORY このフラグを使用して、指定した名前がファイル名ではなく、すでにメモリにロードした Bink ファイル全体へのポインタであることを Bink に通知します。Bink は、このメモリを再生の開始位置として使用します。BinkClose を呼び出すまでは、渡したメモリ アドレスを解放してはなりません。ビッグエンディアンのマシンでは、最初にメモリ バッファに対して DWORD 型でエンディアンのスワップ処理を行ってから、BinkOpen に渡す必要があります。
 BINKNOFRAMEBUFFERS このフラグを使用して、フレーム バッファを手動で割り当てるため、内部ビデオ フレーム バッファを割り当てないように Bink に指示します。このフラグと BinkGetFrameBuffersInfo 関数を使用して、必要な割り当てに関する詳細情報を取得できます。独自のフレーム バッファを割り当てたら、BinkRegisterFrameBuffers を呼び出して、フレームの解凍先を Bink に指示します。通常、このフラグは、Bink ムービーを直接ソニー PS3、ソニー PSP、Wii、ゲームキューブ、Xbox、または Xbox 360 テクスチャに解凍するために使用します。これにより、ピクセル シェーダ/レジスタ コンバイナを使用して、YcRcB データを直接 RGB ピクセルに変換できます。
 BINKSNDTRACK このフラグを使用して、指定したサウンド トラックを再生するように Bink に指示します。デフォルトでは、Bink は Bink ファイルの最初のサウンド トラックを再生します。このフラグを使用すると、Bink は直前の BinkSetSoundTrack 関数の呼び出しで設定されたトラック ID を使用します。そのため、このフラグを使用する場合は、まず適切なトラック ID の数値を指定して BinkSetSoundTrack を呼び出し、次にこのフラグを使用して Bink ファイルを開く必要があります。
 BINKNOSKIP ビデオがオーディオより遅れ始めると、Bink はフレームをスキップします。Bink ファイルにオーディオがない場合、またはこのフラグを使用した場合は、Bink はフレームをスキップしません。このフラグは、ビデオの品質を音声の品質よりも優先させる場合に便利です (アニメ向き)。
 BINKPRELOADALL このフラグを使用して、ビデオ全体をメモリにロードするように Bink に指示します。Bink がビデオをメモリに読み込めない場合は、ファイルを開こうとしても失敗します。
 BINKALPHA このフラグは、ファイルにアルファ平面のデータがあれば解凍するように Bink に指示します。デフォルトでは Bink はアルファ平面を解凍しないので、アルファ ビット平面のフォーマット (BINKSURFACE32A または BINKSURFACE4444) を使用する場合は、このフラグを設定する必要があります。
 BINKGRAYSCALE このフラグは、グレースケールで Bink ファイルを再生するように Bink に指示します。解凍処理 (Bink は Y 平面だけを解凍) でも YUV 変換処理でも、グレースケールのモードのほうがはるかに高速です。Bink ファイルをグレースケール モードで圧縮する場合は、自動的にこのフラグが設定されます。
 BINKFRAMERATE このフラグを使用して、Bink ファイルのフレーム レートを上書きして、直前に呼び出した BinkSetFrameRate 関数で指定したフレーム レートを使用するように Bink に指示します。そのため、このフラグを使用する場合は、まず適切な比率を指定して BinkSetFrameRate を呼び出し、次にこのフラグを使用して Bink ファイルを開きます。
 BINKCOPY2XH このフラグを使用して、高さ 2 倍モードでビデオを再生するように Bink に指示します。このフラグで、それぞれの走査線を (高さを 2 倍に拡大して) 複製します。Bink コンプレッサを使用してビデオを高さ 2 倍モードで圧縮すると、このフラグは自動的に設定されますが、他の COPY フラグで上書きできます。
 BINKCOPY2XHI このフラグを使用して、高さ 2 倍インターレース モードでビデオを再生するように Bink に指示します。このフラグでビデオを 1 つおきの走査線に拡大します (高さが 2 倍になります)。Bink コンプレッサを使用してビデオをインターレース モードで圧縮すると、このフラグは自動的に設定されますが、他の COPY フラグで上書きできます。
 BINKCOPY2XW このフラグを使用して、幅 2 倍モードでビデオを再生するように Bink に指示します。このフラグで、それぞれのピクセルを (幅を 2 倍に拡大して) 複製します。Bink コンプレッサを使用してビデオを幅 2 倍モードで圧縮すると、このフラグは自動的に設定されますが、他の COPY フラグで上書きできます。
 BINKCOPY2XWH このフラグを使用して、幅および高さ 2 倍モードでビデオを再生するように Bink に指示します。その結果、すべての走査線とピクセルが (高さと幅のどちらも 2 倍で) 複製されます。Bink コンプレッサを使用してビデオを幅および高さ 2 倍モードで圧縮すると、このフラグは自動的に設定されますが、他の COPY フラグで上書きできます。
 BINKCOPY2XWHI このフラグを使用して、幅 2 倍、高さインターレース モードでビデオを再生するように Bink に指示します。その結果、各ピクセルは複製され、走査線は 1 本おきにスキップされます (幅と高さの両方とも 2 倍になります)。Bink コンプレッサを使用してビデオを幅 2 倍、高さインターレース モードで圧縮すると、このフラグは自動的に設定されますが、他の COPY フラグで上書きできます。
 BINKCOPY1XI このコピー フラグは、通常の等倍モードでビデオを再生しながら、再生時に走査線を 1 本おきにコピーするように Bink に指示します。フレームのコピーの時間が通常の半分になるので、性能の低いマシンでは便利です。
 BINKCOPYNOSCALING このフラグを使用して、拡大やインターレースを行わずにビデオを再生するように Bink に指示します。このフラグを使用して、インターレース モードまたは拡大モードを使用して圧縮したビデオの効果が自動的に有効にならないようにします。
 BINKNOMMX このフラグは、色空間を変換または解凍している間は MMX を使用しないように Bink に指示します。MMX を使用しないと、Bink の実行速度は大幅に低下します。通常はこのフラグを変更しないでください。Xbox では、このフラグをサポートしていません。Xbox で実行する場合は常に MMX が使用されます。
 BINKSIMULATE このフラグを使用して、直前に呼び出された BinkSetSimulate 関数で設定されたデバイスの速度をシミュレートするように Bink に指示します。このフラグを使用する場合は、まず適切な速度を指定して BinkSetSimulate を呼び出し、次にこのフラグを使用して Bink ファイルを開きます。
 BINKIOSIZE このフラグを使用して、直前に呼び出された BinkSetIOSize 関数で設定されたサイズでデフォルトの I/O バッファ サイズを上書きするように Bink に指示します。このフラグを使用する場合は、まず I/O のサイズを指定して BinkSetIOSize を呼び出し、次にこのフラグを使用して Bink ファイルを開く必要があります。
 BINKNOFILLIOBUF このフラグを使用して、BinkOpen で I/O バッファに読み込まないように Bink に指示します。デフォルトでは、ファイル I/O バッファがいっぱいになってから BinkOpen から戻ります。このフラグを使用するとファイルを開く処理は速くなりますが、ビデオのスキップが早めに発生する可能性があります。
 BINKIOPROCESSOR このフラグにより、Bink が Bink ファイルの読み取りに使用するファイル I/O システムを完全に置き換えます。これには手間がかかりますが、このフラグを使用するとすべての種類の I/O を (インターネット アドレスからのストリーミングも) 実行できます。このフラグの使用に関する詳細については、RAD にお問い合わせください。
 BINKNOTHREADEDIO このフラグを使用して、マルチスレッドの I/O を使用しないように Bink に指示します。デフォルトでは、Bink はバックグラウンド スレッドを作成して、Bink ファイルをバックグラウンドで読み込みます。Bink がバックグラウンド処理をしないように指定する場合は、このフラグでオフにできます。

説明

Bink フラグは、特殊なモードでの処理を有効にする場合に BinkOpen 関数に渡されます。ほとんどの場合は Bink フラグとして 0 を渡しますが、必要であればこれらのオプションを使用できます。

さまざまな COPY フラグにより、非常にスケーラブルなビデオ再生を実現できます。たとえば、320x240 のムービーがあるとき、各種の COPY フラグを使用して次のような再生モードを提供できます。

1) フラグなし - 標準の 320x240 で再生

2) BINKCOPY1XI - 低速マシン用の等倍インターレースの 320x240。

3) BINKCOPY2XWHI - 中速マシン用の 2 倍インターレースの 640x480。

4) BINKCOPY2XWH - 高速マシン用の 2 倍拡大の 640x480。
