GNU/Linux での Bink ビデオ フレームの描画

説明

GNU/Linux でのフルカラーのビデオ再生は、ディストリビューションによって大きく異なります。Bink はどのブリッティングのシステムでも動作しますが、ここで使用するサンプルでは常に SDL グラフィック ライブラリを使用しています。SDL を提供するディストリビューションは多く、SDL を使用するゲームも多いので、エンドユーザーのハードウェアでもすでに実行されている可能性があります。

フルカラーを扱う通常のコーデックと同様に、Bink は RGB 色空間ではなく、YUV 色空間を使用します。このとき必要になる色空間の変換を高速で安定して実行するために、Bink には YUV を RGB にソフトウェア変換する最適化されたルーチンが組み込まれています。

幅を等倍や 2 倍にするルーチン、高さのインターレースのルーチン、高さを 2 倍にするルーチン、アルファ平面マージのルーチン、16 ビット、32 ビットの RGB フォーマットにコピーできるグレースケールまたはカラーのルーチンなどがあります。手作業で最適化された合計 80 種類のルーチンが用意されています。Bink のライブラリの半数以上は、色空間の変換用です。

SDL や単純なブリッティング API のインターフェイスを開く方法については、サンプル プログラムを参照してください。
