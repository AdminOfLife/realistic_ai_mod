マウス カーソル

説明

マウス カーソルは、マウスを動かすと動きます。Windows がクラッシュすると動かなくなります。

マウス カーソルには、ハードウェアとソフトウェアの 2 種類があります。ハードウェア カーソルは、画面のピクセル上にフロートしていて、他の要素に影響しません。一部のビデオ カードは、白黒の場合に限ってハードウェア カーソルを表示できます。また一部のビデオ カードは、非アニメーションのカーソルだけを提供しています。一方、すべてのタイプのカーソルを扱えるビデオ カードもあります。ハードウェア カーソルをまったく提供できないビデオ カードもあります。

ハードウェアが特定のタイプのカーソルを作成できない場合は、Windows または MacOS がソフトウェア カーソルを作成します。ソフトウェア カーソルは、実際にはシステムによって描画され、マウスを移動するとシステムが古いマウスのビットマップを消去して、数ピクセル先に新しいマウスを描画します。

通常、このことを知る必要はまったくなく、それでもすべてが問題なく動作します。ただし、Windows の DirectDraw または MacOS では、オペレーティング システムに認識されないままビデオ メモリに書き込むことができます。したがって、ユーザーがビデオ メモリに書き込んでからソフトウェア カーソル システムのマウスを移動した場合、Windows または MacOS が古いマウスのビットマップを消去するために移動前の場所に描画するピクセルは、不適切なピクセルになります。これによって残る小さなごみが、いわゆるマウスの軌跡です。

マウスの軌跡を避けるには、DirectDraw または MacOS pixmap サーフェスをロックする前にマウスを非表示にし、このサーフェスをロック解除してから復元します。ただし、これによってビデオ再生中にわずかながらフリッカが発生するため、ソフトウェア カーソルを検出して、マウスの軌跡が生じる場合にだけカーソルを非表示にすることをお勧めします。

Bink には、この問題に対処するために BinkIsSoftwareCursor、BinkCheckCursor、および BinkRestoreCursor 関数が用意されています。これらの関数は、DirectDraw または MacOS を使用する場合はいつでも、さらに Bink を使用しない場合でも役立ちます。
